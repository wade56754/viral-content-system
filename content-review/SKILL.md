# content-review v2.4 — 内容质量门禁

> 版本：v2.4 | 更新：2026-03-22
> 从 v1.0 重构：拆分 references/ + 新增进化系统 + 入口路由器

---

## 第一性原理

**审稿只回答一个问题：这篇内容发出去，读者会不会停下来、看完、并且做点什么？**

停下来 = Hook + 标题。看完 = 结构 + 节奏 + 人味儿。做点什么 = 行动引导 + 争议性。
三个环节任一断裂，内容就失败了。评分体系围绕这三个环节展开。

---

## 入口路由器

| 输入 | 路由 | 说明 |
|------|------|------|
| 被写作 Skill 的 Phase 7 调用 | → 完整三关审稿 | 标准流程 |
| 用户发了一篇内容说"帮我看看" | → 完整三关审稿 | 同上 |
| 用户说"这篇能发吗" | → 快速版：硬门槛 + 评分，跳过16项质控 | 只要结论 |
| 用户说"帮我改" | → 不走审稿，建议转写作 Skill 的改稿模式 | 审稿≠改稿 |
| 用户给了多篇内容 | → 逐篇审，不批量打分 | 每篇独立评分卡 |
| platform = x-thread / x-short / x-article | → **X 推特门禁模式** | 只说过/不过，不给改法 |

**⚠️ 审稿 ≠ 改稿。content-review 只出评分，不动内容。**
**⚠️ X 推特门禁模式：不合格只说"重写"，不给具体改法。逼写手自己想。**

---

## 审稿铁律（全程有效）

1. **三关顺序不可跳** — 硬门槛 → 质控 → 评分。第一关不过禁止进第二关
2. **按维度打分，不凭感觉** — 每个维度有4档细则，严格对照打分
3. **X 推特门禁模式：不给改法** — platform 为 x-* 时，不合格只说"重写"，不输出任何改进建议
4. **非 X 平台：改进建议必须具体** — "Hook可以更强"不是建议。"第一句换成：'7年3次转型1个发现'" 才是
5. **亮点至少1条**（非门禁模式） — 纯负面反馈会打击写手改稿意愿
6. **审稿前必须加载灵魂文件** — 不读 IDENTITY/VOICE_DNA 就审 = 标准不一致

## Phase 工作流

### Phase 1：硬门槛（任一不过 → KILL，不进后续）

读 `references/platform-rules.md` 确认平台特定规则。

| # | 检查项 | 适用 | 不过后果 |
|---|--------|------|---------|
| H1 | IDENTITY 宪法冲突？ | 全部 | KILL |
| H2 | VOICE_DNA 禁区违反？ | 全部 | KILL |
| H3 | **写作基因宪法检查**（见下方） | 全部 | 违反任一条=KILL |
| H4 | AI辅助标注？ | xhs | 未标注=KILL |
| H5 | 字数在平台限制内？ | 全部 | 打回修改 |
| H6 | 外链位置合规？ | x-* | 正文有外链=打回 |
| H7 | Hashtag数量合规？ | 全部 | 超量=打回 |

#### H3 — 10条写作基因逐条对照

> 来源：你的 SOUL.md §写作基因
> 这10条高于任何模板、公式、平台规则。违反任一条 = KILL，无例外。

| # | 基因 | 检测方法 | KILL 条件 |
|---|------|---------|----------|
| G1 | 第一句话让人读第二句 | 第一句是否短、有力、制造好奇？ | 第一句>15字且无钩子 |
| G2 | 一篇只打一个钉子 | 全文是否围绕一个核心论点？ | 有≥2个不相关核心论点 |
| G3 | 删减——写完删30% | 有无冗余段落/重复论述/可删不影响的句子？ | 明显臃肿（≥3处可删段落） |
| G4 | 名词动词驱动，形容词是脂肪 | 抽查5个句子，驱动力是名词动词还是形容词堆砌？ | ≥3句靠形容词撑场 |
| G5 | 具体细节 > 抽象概念 | 有无数字/场景/案例？ | 全篇零具体细节（纯抽象） |
| G6 | 每段有价值翻转 | 段首和段尾是否有认知位移？ | ≥50%段落无翻转（纯平铺） |
| G7 | 结论先行 | 核心观点在前1/3出现？ | 核心观点藏在最后才揭晓 |
| G8 | 人味是终极护城河 | 有无"我"的经历/错误/语气？ | 全篇零个人痕迹（纯说教） |
| G9 | 重写≥3遍 | （无法检测，跳过） | — |
| G10 | 该停就停 | 结尾是否在啰嗦总结已说过的话？ | 结尾重复正文核心观点≥2句 |

**执行方式：** 逐条对照，命中任一 KILL 条件 → 输出违反的基因编号 + 一句话说明，终止审稿。
**G9 无法从成品检测，标记为"默认通过"。**

**第一关全过（H1-H7 + G1-G10）→ 进 Phase 2。任一不过 → 输出 KILL 原因，结束。**

### Phase 2：16项质控清单

读 `references/anti-patterns.md` 检测反面教材。

**必过项（8项 — 任一未过强制改完才进评分）：**

| # | 检查项 |
|---|--------|
| 1 | 标题有数字/情绪词/反常识？ |
| 2 | 前2-3行有画面感钩子？ |
| 3 | 有利益承诺（读完能得到什么）？ |
| 4 | 有≥3处可复制素材？（长文类） |
| 5 | 结尾有行动引导？ |
| 6 | 呼吸感：≥5处关键句独占一行？ |
| 7 | 核心关键词前置？ |
| 8 | **论据有出处？**（核心论据来自知识库/弹药库/个人经验/公认事实，非AI编造？有无出处不明的数据/案例/对比？） |

**重要项（6项）：**
认知框架 / 价值论证 / 人味儿 / 独特洞察≥2 / 字数最优区间 / **故事融合质量**（x-article/x-thread）

**故事融合检查（仅 x-article/x-thread 平台）：**

| # | 检查项 | 不过表现 |
|---|--------|---------|
| S-1 | 论点是否出现在故事之前？ | 先说观点再"比如"→ 标记 |
| S-2 | 故事有无读者误判/转折？ | 平铺直叙无反转 → 标记 |
| S-3 | 故事结尾是金句接住还是"这说明"式总结？ | "这个故事告诉我们" → 标记 |
| S-4 | 故事有无无关但有画面感的细节？ | 全是论点服务性细节 → 标记 |

命中≥2条 → 故事融合失败，人味儿维度扣3分，备注"故事退化为例子"

**加分项（4项）：**
自检清单 / 数据对比 / 情绪节奏完整 / 结尾有留白

**第二关判定：必过项全过 → 进 Phase 3。未过 → 列出未过项，要求改完重审。**

### Phase 3：100分评分

读 `references/scoring-system.md` 按维度逐项打分。

**通用维度（75分）：**

| 维度 | 权重 | 核心问题 |
|------|------|---------|
| Hook强度 | 20分 | 前3行能让人停下来？ |
| 标题爆发力 | 15分 | 不得不点？ |
| 痛点具象化 | 15分 | 有数字+场景？ |
| 🫁 人味儿 | 10分 | 像人写的？ |
| 争议程度 | 10分 | 会引发讨论？ |
| 行动引导 | 5分 | CTA有效？ |

**平台特化维度（25分）** → 详见 `references/scoring-system.md`

**发布门槛：**
```
≥ 80分 → ✅ 可发布
65-79分 → ⚠️ 有条件发布（改完指定项重审）
< 65分 → ❌ 打回重写
```

**维度最低线（任一不达标 → 打回，无论总分）：**

| 维度 | 最低线 | 理由 |
|------|--------|------|
| Hook强度 | < 10分 | 开头救不了 |
| 标题爆发力 | < 6分 | 没人点开 |
| 即时可用性 | < 8分 | 纯说教（X长文） |
| SEO命中度 | < 4分 | 没搜索流量（小红书） |
| 人味儿 | < 5分 | AI味太重 |

### Phase 4：输出评分卡

按 `templates/scorecard.md` 格式输出完整评分卡。

## ⚠️ Gotchas（审稿踩坑）

> 完整版见 `data/gotchas.md`

1. **自己审自己容易放水** — 严格按维度细则，不凭"整体感觉"
2. **人味儿最难量化** — 先跑 anti-patterns 24项检测，命中数就是客观分
3. **改进建议太泛没用** — 必须给具体改法，不是"可以更好"
4. **硬门槛容易被跳过** — 三关顺序写死，第一关不完成禁止进第二关

5. gotchas 中 ≥3 次的模式 → 晋升到铁律
