## 🧠 核心框架

### 裂隙的 4 种形状

| 裂隙类型 | 读者的缺口 | 撕开方式 |
|---------|-----------|---------|
| 🧠 认知裂隙 | "我以为的是错的" | 反常识炸弹 / 因果倒置 |
| 🔧 能力裂隙 | "我不会做这件事" | 痛点具象化 / 极端案例 |
| 💡 信息裂隙 | "我不知道这个东西" | WOW 事实 / 稀缺信息 |
| ❤️ 情感裂隙 | "原来别人也这样" | 个人故事 / 脆弱时刻 |

### 3 Pass 执行制

```
Pass 1（骨架 + 弹药 + 初稿）
  → 写作铁律 + 裂隙框架 + 武器库填弹药

Pass 2（风格化 + 去 AI 化）
  → 风格 DNA 改写 + 24 模式 AI 味检测

Pass 3（平台优化 + 发布包）
  → Hook Tweet × 3 + 封面 + 金句卡片
```

### 9 阶段完整工作流

```
Phase 0: 灵魂加载（身份/风格/受众/规则/踩坑清单）
Phase 1: 找谜题（选题质检 — 三问门槛）
Phase 2: 骨架设计（论点→论据→比喻→自我反驳）
Phase 3: 弹药检索（武器库匹配）
Phase 4: 写作（共创模式 / AI 主导模式）
Phase 5: 风格化 + 去 AI 化（24 项检测）
Phase 6: 自检（8 项硬指标）
Phase 7: 独立审稿（100 分评分体系，≥75 分才放行）
Phase 8: 发布包生成（封面/金句卡片/Hook Tweet）
Phase 9: 评论区运营 + 数据反馈 + 进化闭环
```

## 🔑 7 条写作铁律

1. **犹豫就删** — 删减是最核心的写作能力
2. **第一句话唯一任务是让人读第二句话** — 开头决定生死
3. **具体 > 抽象** — 数字和场景比形容词强 100 倍
4. **每句话过"读者关心吗"测试** — 读者不关心的删掉
5. **重写才是真正的写作** — 第一稿只是思考过程
6. **弹药是树叶不是树根** — 删掉所有弹药骨架还站得住
7. **技术是佐证，认知才是正文** — 读者要的不是你怎么做，是你明白了什么

## 📈 进化系统

系统内置闭环进化机制：

- **写前**：加载踩坑清单 + 改稿规律 + 已用弹药（防重复）
- **写后**：原稿存档 → diff 原稿 vs 终稿 → 提炼新踩坑 → 更新规律
- **月度**：踩坑 ≥3 次的规律 → 晋升为写作铁律
